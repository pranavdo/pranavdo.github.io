Gitlet

Berkeley

Implemented my own version-control system that mimics the basic features of Git including init, add, commit, rm, log, global-log, find, status, checkout, branch, rm-branch, reset, and merge. The internal structure of the software included blobs (saved content of the files), trees (directory structures mapping names to references to blobs and other trees), and commits (combinations of log messages, other metadata (commit date, author, etc.), a reference to a tree, and references to parent commits). Used cryptographic hash functions like SHA-1, serialization, directory manipulation, graph traversals, and integration tests. The application was built with Java.

Scheme Interpreter

Berkeley

Implemented a Scheme interpreter built with Python. Using the REPL (read, evaluate, print, loop) method, the scheme code is first parsed through lexical and syntactic analysis using mutually recursive functions. Then the next step is the evaluation of the tokenized input and calling specific functions depending on the code. The obtained value is then printed and the loop continues allowing for the user to enter in another line of Scheme code.

brOSKI

CalHacks 2023

brOSKI is a chatbot engineered to emulate a UC Berkeley student, offering guidance through campus life by tapping into the collective intelligence of the Berkeley Subreddit. It navigates Reddit's API restrictions by delving into Pushshift's 20TB archive, extracting 150GB of r/Berkeley posts. This targeted approach, initially omitting comments, later integrated them through post ID queries. The chatbot's core, powered by the fine-tuned LLaMA 2 13b model with 3500 Q&A samples, was meticulously optimized for authentic, unbiased interactions. While the full-fledged web interface is forthcoming, the current front end, designed with Figma and implemented with HTML and CSS, connects seamlessly to the backend via API, establishing a digital embodiment of a Berkeley student advisor.

Project Water

Datathon For Social Good 2022 - Winner

The impact of legislative policies addressing water consumption in drought-prone California was assessed to understand the role of public policies in resource management. Large datasets, comprising over 40,000 entries, were modeled to test whether the median potable water consumption in each Californian county reduced by at least 15% between 2020 and 2022, in line with the targets set by Executive Order N-10-21. Data visualizations were generated to identify trends in confounding variables such as potable water production. Statistical tests, including sharp null hypothesis testing (A/B testing), were conducted to create metrics, and outlier analysis was performed to establish baselines.
